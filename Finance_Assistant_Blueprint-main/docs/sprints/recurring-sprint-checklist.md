# Recurring Sprint Checklist

Use this checklist at the start and end of every sprint.

## Sprint setup
- [ ] Create sprint planning note in `docs/sprints/` (scope, why, exit criteria)
- [ ] Confirm project board statuses (Backlog / In Progress / Done)
- [ ] Confirm dependencies and target dates for active sprint items

## During sprint
- [ ] Update item fields: Sprint, Phase, Owner, Target Date, Exit Criteria
- [ ] Track blocked work and explicit dependency reasons

## Sprint close
- [ ] Publish sprint retro note in `docs/sprints/` (shipped, slipped, lessons learned)
- [ ] Update `CHANGELOG.md` with sprint outcomes
- [ ] Validate milestone acceptance checks impacted by this sprint

## Cadence policy
- [ ] Enforce 2-week sprint cycle
- [ ] Schedule 1-week buffer after every 3 sprints
